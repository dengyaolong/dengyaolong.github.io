无污染无添加的restful

发表于 2016-10-28 |

域名

  1. 尽量将api部署在专用域名下，如：https://api.example.com
  2. 如果带版本，可以加入版本号，如：https://api.example.com/v1/

URI

  1. 使用复数名词表示资源集合
  2. 尽量不要出现动词。如 /users/add，使用 POST /users替代

    Method

    CRUD

    GET：查询资源
请求 含义
GET /users 获取用户信息列表
GET /users/1 获取id=1的用户信息
GET /users/1/friends 获取id=1的用户的朋友列表

POST: 创建资源

请求 含义
POST /users 添加用户
POST /users/1/friends 给id=1的用户添加好友

PUT: 更新资源

请求 含义
PUT /users/1 更新id=1的用户
PUT /users/1/friends/2 修改id=1的用户的id=2的好友的信息

DELETE: 删除资源

请求 含义
DELETE /users/1 删除id=1的用户

安全性与幂等性

  • 安全性：不会改变资源
  • 幂等性： 每次执行对资源状态改变效果是等价的。
Method 安全性 幂等性
GET √ √
POST × ×
PUT × √
DELETE × √

条件过滤

在querystring中加入条件过滤, 常见的有

  • ?limit=20，限制头20条
  • ?offset=10，偏移10条
  • ?order=asc&sortBy=x，按x顺序排
  • ?x=1，获取x=1的数据

返回合适的状态码

状态码 含义
200 GET请求成功
201 POST同步请求完成
202 POST, DELETE异步请求接受
204 无返回，一般用于DELETE请求成功
206 GET请求成功，但返回只是部分结果
401 用户没有认证（密码错误，令牌错误）
403 用户已经认证，但是请求被禁止，比如权限问题/黑名单
404 请求资源不存在
405 请求方法不允许
409 冲突导致的请求失败，一般用于后端处理数据时候约束冲突

更多看https://www.w3.org/Protocols/rfc2616/rfc2616-sec10.html

返回结果

获取资源列表，尽量都用JSON

请求 返回 请求示例 返回示例
GET /resources 资源对象数组（列表） GET /users 
[
  {id: 1,name: ‘a’}, 
  {id: 2,name: ‘b’}
]
GET /resources/:id 单个资源对象 GET /users/1 
{
  id: 1,
  name: ‘a’
}
POST /resources 创建后的资源对象 POST /users
payload: {name: ‘c’}		 
{
  id: 3,
  name: ‘c’
}
PUT /resources/:id 修改后的资源对象 PUT /users/3
payload: {name: ‘cc’}		 
{
  id: 3,
  name: ‘cc’
}
DELETE /resources/:id 空，状态码为204 DELETE /users/3 空

错误处理

出现错误返回出错信息，以及对应的状态码。

这里建议是：

  • body.message是有用的错误信息
  • body.code是具体业务前后端约定的一些错误标识
